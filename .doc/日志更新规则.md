## 更新日志维护规则

1. 目标与范围
- 维护 `CHANGELOG.md`，记录每个版本变更。
- 格式遵循语义化版本（SemVer）；当前项目采用“Version 标题 + emoji 类别”的样式。
- 严禁直接使用 Git/SVN 命令；仅使用项目脚本或 CLI。

2. 获取日志
- 推荐使用命令：`npm run changelog x.x.x`（调用 `hy-frame changelog x.x.x`），自动生成规范条目并可写入至 `CHANGELOG.md` 顶部。
- 仓库地址与版本源信息以 `package.json` 的 `repository.url` 与 `config.svnUrl` 为准，无需手动执行 `svn info`。
- 脚本会优先从 `projects/frame/package.json` 检测版本文件，若目录变更为 `packages/frame/package.json` 将自动适配。

3. 版本边界判定
- 以“修改/更新版本号为 x.y.z”的提交为最新版本起点，以上一版本对应提交为终点，按区间汇总本次变更。
- 日期以“更新版本号为 x.y.z”的提交日期为准，格式 `YYYY-MM-DD`；最终在标题中以斜体括号形式呈现。
- 遇到无法解析版本提交的情况，按 `CHANGELOG.md` 中相邻版本日期区间回退解析。

4. 编写格式
- 顶部标题：`# 📦 更新日志`。
- 版本标题：`## 🚀 Version x.y.z _(YYYY-MM-DD)_`。
- 版本分隔：相邻版本之间使用一行 `---` 分隔。
- 类别顺序与命名：
  - `### ✨ 新增功能`
  - `### 🛠️ 修复`
  - `### 🔧 变更`（如为破坏性变更，尾部标注“（破坏性变更）”）
  - 如有需要再添加：`### 🗑️ Removed`、`### 🕒 废弃`、`### 🔐 安全`
- 标识符统一使用反引号包裹：组件/指令/属性/方法/事件/类型，如 `hy-list`、`onPullDownRefreshStart`、`HyHttpOptions`、`boolean`。
- 中文描述简洁明确，不写实现细节或提交人信息。

5. 分类映射规则
- 新增功能：含“新增/添加/支持/引入”；用于新组件、指令、属性、事件、管道、用法、功能。
- 修复：含“修复/解决/纠正/异常/错误/报错”；用于样式、国际化、交互、逻辑缺陷。
- 变更：含“调整/优化/改进/更新/重命名/统一命名/还原/回退”；非破坏性行为。破坏性变更需在条目末尾标注“（破坏性变更）”。
- Removed：含“移除/删除”；已移除的功能，不应再使用。
- 废弃：含“废弃/弃用”；仍可用但计划移除的功能。
- 安全：含“安全/修复漏洞”；修复了安全问题的变更。

6. 工作流程
- 使用 `npm run changelog x.x.x` 获取最近版本修订及区间。
- 按版本边界截取提交区间，避免越界；对重复或噪声信息进行合并与归类。
- 提炼并规范化描述，统一术语与命名，补充必要上下文（组件、属性、类型、默认值）。
- 按分类映射分配到“新增功能/修复/变更/Removed/废弃/安全”。
- 写入 `CHANGELOG.md` 顶部，保持现有样式与顺序。
- 自检：类别顺序正确、标识符加反引号、日期与版本边界准确、中文描述精炼、相邻版本间有 `---` 分隔。

7. 破坏性变更处理
- 出现在“变更”类别中，明确标注“（破坏性变更）”，必要时说明旧名 → 新名与迁移提示。

8. 禁止事项
- 不直接使用 Git 或 `svn log`/`svn info` 手工获取日志；仅使用项目提供的脚本或 CLI。
- 不写提交人、提交号、文件路径等实现细节。
- 不并入非该版本边界内的提交。
- 不随意更改历史版本的语义归类与格式（除纠错）。

9. 示例模板
```
## 🚀 Version x.y.z _(YYYY-MM-DD)_

### ✨ 新增功能
- 新增 `hy-xxx` 组件/指令/属性 `propName`，类型：`Type`，默认值：`Default`，说明：用途描述
- 新增 `hy-list` 事件 `onPullDownRefreshStart`，说明：触发下拉刷新，请求开始

### 🛠️ 修复
- 修复 `hy-yyy` 在某场景下的样式/交互/报错问题（问题简述）

### 🔧 变更
- 调整 `hy-zzz` 方法/属性命名：`oldName` → `newName`
- `hy-i18n` 路由类型限制为 `string[]`（破坏性变更）

### 🗑️ Removed
- 删除 `certificateCrtName`、`certificateKeyName`

---
```
